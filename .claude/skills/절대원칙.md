# 1080사찰 대작불사 — 절대 원칙

## 사찰 불가침 원칙
- 문수사(munsusa) 🔒 절대 수정 금지
- 보림사(borimsa) 🔒 절대 수정 금지
- 모든 일괄 스크립트에서 위 두 slug 반드시 제외
- 위반 시 critical incident

## 코드 작업 원칙
- 수정 전 반드시 실제 파일 내용 확인 (추측 코드 생성 금지)
- 기존 코드·파일 함부로 삭제 금지
- 지시하지 않은 기능·텍스트 임의 추가 절대 금지
- 코드 생성 시 항상 파일 배치 경로 명시
- 코드 생성 시 항상 git push 명령어 함께 제시

## 렌더링 원칙
- UI 반영만으로는 불충분 — 반드시 실제 사찰 URL 렌더링 검증
- 블록보물함 작업 중 코드 수정은 선발대 20개 사찰 기준 테스트
- 1080사찰 전체 일괄 적용은 대표님 별도 지시 후 실행
