## 이상 발견 분석

### 주1: API-2, API-3 — Origin 차단/slug 검증 미동작

**원인**: `requireTempleAuth` 패치(세션 2A, 커밋 386b78e)는 **feature/rls-poc 브랜치**에만 존재. 프로덕션(main)에는 아직 미병합.

**현재 프로덕션 상태**: offering/route.ts에 Origin 체크·slug 존재 검증 없음.
**Phase 2 main 병합 시**: 이 2건이 403/404로 전환될 예정.
**리스크**: Phase 1 동결 기간이므로 의도된 상태. 즉각 조치 불필요.

### 주2: 경로 5 — admin 404

**원인**: `miraesa.k-buddhism.kr/admin` → 미들웨어가 `/miraesa/admin`으로 rewrite → `app/[slug]/admin` 라우트 미존재 → 404.
**정상 경로**: `temple-admin-zeta.vercel.app/admin/miraesa` (Vercel 기본 도메인 경유).
**영향**: 없음. 서브도메인 경유 admin 접근은 원래 지원 안 함.

### 주3: 경로 6 — believers 400

**원인**: 프로덕션 believers/route.ts는 `requireTempleAuth` 미적용 (feature/rls-poc에만 존재). `temple_slug` 파라미터 없이 호출 → 기존 400 ("temple_slug 필수").
**Phase 2 병합 후**: 401/403으로 전환 예정.
